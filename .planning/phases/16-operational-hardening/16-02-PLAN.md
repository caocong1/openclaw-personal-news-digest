   - `pattern`: a regex string (case-insensitive)
   - `priority`: integer (lower = higher priority)
   - `example`: a string example of a matching title

   Include exactly these starter patterns:
   - `^Top \\d+` (priority 1, example: "Top 10 AI Papers of 2025")
   - `^\\d+ Best` (priority 1, example: "5 Best Open Source LLMs")
   - `weekly roundup` (priority 2, example: "AI Weekly Roundup: Feb 2025")
   - `monthly digest` (priority 2, example: "February Monthly AI Digest")
   - ` \\d+ articles?` (priority 3, example: "10 Articles Every Developer Must Read")
   - `\\d+ links?` (priority 3, example: "7 Links Worth Your Attention")
   - `highlights of (the )?week` (priority 3, example: "Highlights of the Week in Tech")
   - `everything you need to know` (priority 4, example: "Everything You Need to Know About GPT-5")
   - `summary of` (priority 4, example: "Summary of Latest LLM Research")
   - `collection of` (priority 4, example: "Collection of AI Safety Resources")
   - `curated (list|links|picks)` (priority 4, example: "Curated List of AI Tools")

   - The LLM should also set `is_roundup` on the output object.
   - `is_roundup: true` when the item is an aggregation/roundup (no original reporting, summarizes or links to other items).
   - `is_roundup: false` when the item is a standalone report with original content or analysis.
   - `is_roundup: null` only if the LLM cannot determine.
   - When `is_roundup: true`, also populate `roundup_children` with the IDs of any child items that should be created.
   - The LLM should NOT create child items -- it only sets `is_roundup: true` and optionally lists known child URLs in `roundup_children`.

3. Do NOT modify the existing classify output fields (categories, importance_score, form_type, tags, digest_eligible). Do NOT change the JSON output format structure.

   Step 7b. **Atomize roundups**:

   - If `is_roundup: true`, write the atomized child items to the same JSONL file.
