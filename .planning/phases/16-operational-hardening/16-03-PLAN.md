  <name>Task 1: Create append-only run-journal.jsonl with journal_tools.py and run-journal.sh</name>

  <action>
1. Create `scripts/lib/journal_tools.py` with:
   - `JOURNAL_PATH = "data/metrics/run-journal.jsonl"` as a module-level path (relative to base_dir).
   - `def journal_append(run_id: str, severity: str, stage: str, code: str, message: str, hint: str = "", details: dict = None, source_id: str = None) -> None`:
     - `severity` must be one of: "error", "warning", "security", "info"
     - `stage` must be one of: "collection", "processing", "output", "delivery"
     - `code` is an uppercase ERR_CODE string (e.g. "SRC_TIMEOUT", "LLM_FAILURE")
     - Builds entry dict with: ts (UTC ISO8601), run_id, severity, stage, code, message, hint, details, source_id
     - Writes atomically to JOURNAL_PATH using .tmp rename pattern
   - `def journal_query(journal_path: str, run_id: str = None, severity: str = None, stage: str = None, limit: int = 100) -> list[dict]`: Reads and filters the journal.
   - `def journal_summary(journal_path: str, days: int = 7) -> dict`: Returns a summary of entries in the last N days: count by severity, count by stage, unique run_ids.

2. Create `scripts/run-journal.sh`:
   - Usage: `bash scripts/run-journal.sh append <run_id> <severity> <stage> <code> <message> [hint] [source_id]`
   - Also supports: `bash scripts/run-journal.sh query [--run-id RUN_ID] [--severity SEV] [--stage STAGE] [--limit N]`
   - Also supports: `bash scripts/run-journal.sh summary [--days N]`
   - Uses the heredoc Python pattern: `python3 - "$BASE_DIR" <<'PY'` calling `journal_tools` functions.
   - Default BASE_DIR is ".".

3. Touch `data/metrics/run-journal.jsonl` to create the file (empty, for git tracking).

4. Journal schema per entry:
   ```json
   {"ts":"2026-04-03T00:00:00Z","run_id":"run-20260403-1200-abcd","severity":"error","stage":"collection","code":"SRC_TIMEOUT","message":"Source src-36kr timed out after 30s","hint":"Re-enable manually or wait for auto-recovery after 3 consecutive successes","details":{"source_id":"src-36kr","timeout_seconds":30},"source_id":"src-36kr"}
   ```

5. The journal is append-only -- never overwrite existing entries. The .tmp rename pattern ensures atomic writes even on crash.
</action>
