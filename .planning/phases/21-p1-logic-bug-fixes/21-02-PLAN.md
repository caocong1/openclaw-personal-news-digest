  Two targeted edits in scripts/debug_quick_check.py:

  1. **Fix union-find cluster lookup** (lines 529-539): Replace the `for a in alerts:` loop with `for idx, a in enumerate(alerts):` and change `gid = _find(alerts.index(a))` to `gid = _find(idx)`. This eliminates:
     - The O(n^2) linear search per iteration
     - The wrong-cluster bug when two dicts are value-equal (`.index()` always returns the first match)

     The fixed block should be:
     ```python
     _final = []
     _seen_event = set()
     _seen_url = set(state['alerted_urls'])
     for idx, a in enumerate(alerts):
         gid = _find(idx)
         cu = norm_url(a.get('url',''))
         if gid not in _seen_event and cu not in _seen_url:
             _final.append(a)
             _seen_event.add(gid)
             _seen_url.add(cu)
     alerts = _final
     ```

  2. **Add dollar-anchor guard** in `_same_event()` (line 505): Replace the dollar-only merge line:
     ```python
     if any(x.startswith('$') for x in shared): return True
     ```
     with a guard requiring a second non-dollar anchor:
     ```python
     dollar = [x for x in shared if x.startswith('$')]
     non_dollar = [x for x in shared if not x.startswith('$')]
     if dollar and non_dollar: return True
     ```
     This means a shared dollar amount like "$1B" alone is NOT enough to merge. There must be at least one additional shared non-dollar anchor (company name, codename, phrase, etc.) to confirm the events are related.
  </action>
