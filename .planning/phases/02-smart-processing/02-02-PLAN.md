```json

  "first_seen": "ISO8601",

      "brief": "string (one sentence describing this news in event context)"
    }
  ],
  "_schema_v": 2
}
```

   Field notes: status transitions (active -> stable after 3 days no update, stable -> archived after 7 days), importance updated to max of linked items, keywords used for Step 2 quick match.

   Defaults for missing fields: keywords = [], timeline = [], item_ids = [].

3. Add `## Section 1C: Event Merging (EVT-01)` to processing-instructions.md after Section 1B (Multi-Language Processing). Document the 3-step funnel from 02-RESEARCH.md "Pattern 2: Event Merging Three-Step Funnel":

   **Step 1 -- Topic Pre-filter:**
   - Load data/events/active.json
   - Filter events where event.topic == item.categories.primary
   - Include "stable" events (may reactivate)
   - Expected: 50-200 events total -> 5-20 same-topic candidates

   **Step 2 -- Keyword Quick Match:**
   - Tokenize item title into words
   - Count overlapping tokens with each candidate event.keywords[]
   - Keep candidates with overlap >= 2 tokens
   - Expected: 5-20 -> 1-5 candidates

   **Step 3 -- LLM Precise Merge:**
   - Use references/prompts/merge-event.md with 1-5 candidates
   - Use strong model tier per COST-04 (event merging requires nuanced reasoning)
   - On "merge": add news_id to event.item_ids, add timeline entry, update last_updated, reactivate if stable, update event.importance to max(event.importance, item.importance_score), set item.event_id, set item.dedup_status to "event_merged" if relation != "initial"
   - On "merge" with relation "update"/"correction"/"reversal": also re-summarize event (1 LLM call, strong model) -- EVT-04. Skip re-summarization for "analysis" relation (opinion, not new facts).
   - On "new": create new Event object per schema, append to active.json, set item.event_id
   - Write updated active.json atomically

   **Cross-language event merging (PROC-06):** Unlike title dedup, event merging DOES work cross-language. The LLM prompt handles Chinese and English titles together for merge decisions.

4. Add `## Section 1D: Event Lifecycle Management (EVT-02)` to processing-instructions.md after Section 1C:
   - Run at the START of each pipeline run, before event merging
   - Read data/events/active.json
   - For each event: if status == "active" and (now - last_updated) > 3 days -> set status = "stable"
   - For each event: if status == "stable" and (now - last_updated) > 7 days -> set status = "archived", move to data/events/archived/YYYY-MM.json
   - Create archived directory if needed
   - Write updated active.json atomically
