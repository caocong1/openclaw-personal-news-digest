<objective>
Create `data/provenance/` directory and all 5 artifact files so provenance pipelines can read/write persistent state. Closes DATA-PROV-001 -- uninitialized provenance artifact store that blocks PROV-06, PIPE-01, PIPE-04, DISC-01, PIPE-05 at runtime. Restores the broken E2E flow: Collection -> Provenance -> Source Discovery -> Ranking -> Output.
</objective>

## Gap Context

SKILL.md Bootstrap step 0 (Collection Phase) already lists `data/provenance/` as a required directory, and SKILL.md Provenance Phase step 6 already documents writes to all 5 artifact files. Source Discovery Phase and Output Phase also reference these files by path. However, the directory was never created on disk. The milestone audit confirmed `ls data/provenance/` returns "No such file or directory".

## Tasks

### Task 1: Create directory with .gitkeep

- Create `data/provenance/` directory.
- Place `.gitkeep` in `data/provenance/.gitkeep` so the directory is tracked in git.
- Alternatively, add `data/provenance/` to `.gitignore` if .gitkeep is not desired (existing .gitignore pattern should be checked; the other data/ subdirectories use .gitkeep for archival folders).

### Task 2: Initialize provenance-db.json

Write `data/provenance/provenance-db.json` with an empty ProvenanceRecord store:

```json
{}
```

Schema: object keyed by `NewsItem.id`, per `references/data-models.md` ProvenanceRecord section. Each value is a ProvenanceRecord with fields `id`, `tier`, `tier_confidence`, `tier_source`, `rule_result`, `llm_result`, `original_source_name`, `original_source_url`, `current_source_name`, `current_source_url`, `cited_sources`, `propagation_hops`, `provenance_chain`, `classified_at`, `_schema_v`. Empty store is `{}`.

### Task 3: Initialize citation-graph.json

Write `data/provenance/citation-graph.json` with the canonical empty structure:

```json
{
  "_schema_v": 1,
  "nodes": {},
  "edges": []
}
```

Schema: per `references/data-models.md` CitationGraph section. `nodes` is an object keyed by normalized URL or canonical URL token. `edges` is an appendable array of relationship records.

### Task 4: Initialize tier-stats.json

Write `data/provenance/tier-stats.json` with empty counters:

```json
{
  "_schema_v": 1,
  "days": {},
  "sources": {},
  "last_updated": "2026-04-03"
}
```

Schema: per `references/data-models.md` TierStats section. `days` is keyed by `YYYY-MM-DD`. `sources` is keyed by `source_id`. `last_updated` is ISO8601 date of initialization.

### Task 5: Initialize provenance-discrepancies.jsonl

Write `data/provenance/provenance-discrepancies.jsonl` as an empty file (0 bytes).

Schema: append-only JSONL, each line a ProvenanceDiscrepancyLog record per `references/data-models.md`. Empty file is valid -- readers handle 0-line case.

### Task 6: Initialize discovered-sources.json

Write `data/provenance/discovered-sources.json` with the canonical empty structure:

```json
{
  "_schema_v": 1,
  "sources": [],
  "last_evaluated": "",
  "last_updated": "2026-04-03"
}
```

Schema: per `references/data-models.md` DiscoveredSourcesState section. `sources` is an array of discovered-source records. Empty array is valid for initial state.

### Task 7: Create verification fixture

Create `scripts/verify-provenance-store.sh` -- a shell script that validates all 5 provenance artifact files are readable as valid JSON (or valid JSONL for discrepancies) and match the expected schema shapes. The script should exit 0 if all files pass, exit 1 if any file is missing or malformed.

The script checks:
1. `data/provenance/provenance-db.json` parses as JSON object
2. `data/provenance/citation-graph.json` parses as JSON with `_schema_v`, `nodes`, `edges`
3. `data/provenance/tier-stats.json` parses as JSON with `_schema_v`, `days`, `sources`, `last_updated`
4. `data/provenance/provenance-discrepancies.jsonl` exists and all lines parse as valid JSON
5. `data/provenance/discovered-sources.json` parses as JSON with `_schema_v`, `sources`, `last_evaluated`, `last_updated`

Run the verification script immediately after creating the files to confirm all stores are readable.

## Verification Criteria

1. `ls data/provenance/` lists all 5 artifact files plus `.gitkeep`.
2. `bash scripts/verify-provenance-store.sh` exits with code 0.
3. Each file contains valid JSON (or valid JSONL for discrepancies) matching the schema documented in `references/data-models.md`.
4. The E2E pipeline flow (Collection -> Provenance -> Source Discovery -> Ranking -> Output) can now resolve all provenance store paths at runtime -- no more dead reads.
