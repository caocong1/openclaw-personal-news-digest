<objective>
Reorder file writes so state is persisted before alert is published, preventing duplicate alerts after crash.

Purpose: Currently the script writes ALERT_FILE (line 535) before STATE_FILE (line 549). If the process crashes between these two writes, the alert has been published but the state does not record it — causing the same alert to fire again on the next run. By writing state first, a crash after state-but-before-alert means the alert is lost (acceptable — it won't re-fire), while a crash before state means neither is written (clean retry).

Output: `debug_quick_check.py` with write ordering: (1) STATE_FILE, (2) METRICS_FILE, (3) ALERT_FILE/DIGEST_FILE.
</objective>

```
if MODE == 'quick-check' and selected_alerts:

    ALERT_FILE.write_text('', ...)   # <-- empty alert

```

Change to this flow:

1. Keep the if/elif/else block but REMOVE the ALERT_FILE.write_text / DIGEST_FILE.write_text calls from inside it. Instead, capture the output content in variables:
   - In the `if` branch: set `alert_content = '\n'.join(out).strip() + '\n'`
   - In the `elif` branch: set `alert_content = ''`
   - In the `else` branch: set `digest_content = '\n'.join(digest) + '\n'`

2. After the if/elif/else block, write files in this order:
   ```python
   # CRITICAL ORDER: state first, then metrics, then output files
   # A crash after state-write but before alert-write means the alert
   # is lost (no duplicate on retry). A crash before state-write means
   # clean retry.
   atomic_write_text(STATE_FILE, json.dumps(state, ensure_ascii=False, indent=2) + '\n')
   atomic_write_text(METRICS_FILE, json.dumps(report, ensure_ascii=False, indent=2) + '\n')

   # Output files last — these are the "publish" step
   if MODE == 'quick-check':
       ALERT_FILE.write_text(alert_content, encoding='utf-8')
   else:
       DIGEST_FILE.write_text(digest_content, encoding='utf-8')
   ```

Key details:
- State dict mutations (state['alerted_urls'], state['alerts_sent']) already happen inside the if-block before any writes — no logic change needed there.
- ALERT_FILE and DIGEST_FILE are output files (not state), so they use plain .write_text() — no need for atomic writes on these.
- The comment explaining the ordering is critical for future maintainers.
- Initialize `alert_content = ''` and `digest_content = ''` before the if/elif/else block to avoid NameError on any code path.
