After the directory creation block (after line 22, the `warnings.filterwarnings` line), add a concurrency guard using a non-blocking exclusive file lock:

```python
LOCK_FILE = BASE_DIR / 'data' / '.pipeline.lock'
LOCK_FD = open(LOCK_FILE, 'w')
try:
    fcntl.flock(LOCK_FD, fcntl.LOCK_EX | fcntl.LOCK_NB)
except OSError:
    print('Another pipeline instance is running — exiting.', file=sys.stderr)
    sys.exit(0)
```

Key details:
- Use `fcntl.flock` with `LOCK_EX | LOCK_NB` (exclusive, non-blocking). If lock is held, OSError is raised.
- Exit with code 0 (not an error — second instance gracefully yields).
- The lock file descriptor stays open for the process lifetime — the OS releases it automatically on exit or crash.
- Use a dedicated `.pipeline.lock` file (not `.lock` which is the JSON-based lock from SKILL.md — they serve different purposes).
- `LOCK_FD` must be a module-level variable to prevent garbage collection from closing the fd.

Add `import os, tempfile` to the imports line (line 1 or 2).

Define a helper function after the constants block (after line 28, before the first function definition):

```python
def atomic_write_text(path: Path, text: str) -> None:
    """Write text atomically via tmp-file + fsync + os.replace."""
    fd, tmp = tempfile.mkstemp(dir=path.parent, suffix='.tmp')
    try:
        with os.fdopen(fd, 'w', encoding='utf-8') as f:
            f.write(text)
            f.flush()
            os.fsync(f.fileno())
        os.replace(tmp, str(path))
    except BaseException:
        try:
            os.unlink(tmp)
        except OSError:
            pass
        raise
```

Key details:
- Uses `tempfile.mkstemp` in same directory as target (ensures same filesystem for `os.replace` atomicity).
- `os.fsync` after flush ensures data hits disk before rename.
- `os.replace` is atomic on POSIX (rename within same filesystem).
- On any exception, the tmp file is cleaned up and the original file is untouched.
