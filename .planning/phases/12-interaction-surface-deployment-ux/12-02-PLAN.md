<task type="auto">
  <name>Task 2: Add deterministic recommendation evidence schema, derivation rules, and rendering</name>
  <files>references/data-models.md, references/scoring-formula.md, references/processing-instructions.md, references/output-templates.md, data/fixtures/digest-explainability-sample.json</files>

  <action>
1. In `references/data-models.md`, add an optional `recommendation_evidence` object to the NewsItem schema after `digest_eligible`. Use this exact structure:

```json
"recommendation_evidence": {
  "quota_group": "core|adjacent|hotspot|explore",
  "primary_driver": "topic_match|high_importance|event_followup|diversity_balance|hotspot_injection|exploration_balance",
  "signals": [
    "importance_score=0.85",
    "topic_weight=0.82",
    "source_quality=0.74",
    "repeat_penalty=false"
  ]
}
```

2. In the same file, add field notes stating:
   - `recommendation_evidence` is derived during output selection, not by the summarize prompt
   - `signals` must be concrete key=value strings
   - older records default to `null`

3. In `references/scoring-formula.md`, add a new subsection titled `## Selection Evidence Mapping` that defines the only legal `primary_driver` values and when to use them:
   - `topic_match` for core or adjacent items selected because topic affinity is the dominant factor
   - `high_importance` for items selected primarily on strong importance
   - `event_followup` when event continuity or event boost materially contributed
   - `diversity_balance` when reverse-diversity replacement changed the selection set
   - `hotspot_injection` when Step 6 hotspot injection forced inclusion
   - `exploration_balance` for exploration quota selections

4. In `references/processing-instructions.md`, add a subsection titled `### Selection Evidence Derivation` in the Output Phase area, after quota assignment and before digest assembly. Define exact derivation rules:
   - every selected item gets `recommendation_evidence`
   - derive from actual values already available in scoring and quota allocation
   - include at least three concrete signals:
     - `importance_score=<value>`
     - `quota_group=<value>`
     - `repeat_penalty=<true|false>`
   - when available, also include:
     - `topic_weight=<value>`
     - `event_boost=<value>`
     - `source_quality=<value>`
   - do not use the LLM to generate recommendation evidence text

5. In `references/output-templates.md`, replace the current partial recommendation-reason pattern with a structured evidence line using this exact structure:
   - `入选依据: {primary_driver_label} | 配额: {quota_group} | 证据: {signal_1}; {signal_2}; {signal_3}`

6. Apply that evidence line to all selected item groups in the daily digest template:
   - Core Focus
   - Adjacent Dynamics
   - Today's Hotspot
   - Exploration

7. Create `data/fixtures/digest-explainability-sample.json` with at least four selected items, one for each `quota_group`, each containing a populated `recommendation_evidence` object using the exact field names above.
