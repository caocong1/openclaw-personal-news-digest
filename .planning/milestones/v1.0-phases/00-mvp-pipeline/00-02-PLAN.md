**references/collection-instructions.md** -- Detailed RSS collection and dedup procedure. This document is the detailed reference that SKILL.md's brief Collection Phase steps expand on. Include:

Section 1: RSS Feed Collection
- Use web_fetch with `extractMode: "text"` to fetch RSS XML
- Parse the XML text to extract: title, link, description, pubDate for each <item>
- If XML parsing fails or content is mangled: fall back to exec with `python3 -c "import feedparser; ..."` or similar
- Handle common RSS quirks: CDATA wrapping, missing pubDate (use fetched_at), HTML in description (strip tags for content_snippet)
- Extract language from feed or infer from content (zh/en)

Section 2: URL Normalization Rules
- Strip tracking parameters: remove utm_source, utm_medium, utm_campaign, utm_content, utm_term and any parameter starting with utm_
- Strip other tracking params: fbclid, gclid, ref, source (when clearly tracking)
- Normalize protocol: convert http:// to https://
- Remove www prefix: www.example.com -> example.com
- Remove trailing slash: https://example.com/path/ -> https://example.com/path
- Lowercase hostname: EXAMPLE.com -> example.com
- Compute ID: SHA256(normalized_url)[:16] (first 16 hex chars)
- Compute content_hash: SHA256(title + description stripped of HTML)[:16]

Section 3: Link-Level Dedup
- Read data/news/dedup-index.json (object mapping url_hash -> {news_id, source_id, fetched_at})
- For each new item: compute url_hash from normalized_url
- If url_hash exists in dedup-index -> skip entirely (do not write to JSONL)
- If url_hash not found -> write item to JSONL with dedup_status "unique"
- After all items processed: update dedup-index.json with new entries (atomic write)

Section 4: JSONL Write Format
- File path: data/news/YYYY-MM-DD.jsonl (one file per day, based on fetched_at date)
- Each line: one complete NewsItem JSON object (per references/data-models.md schema)
- New items start with processing_status "raw"
- Use atomic write: write to .tmp.{run_id} file first, then rename

**references/processing-instructions.md** -- Detailed processing, error handling, breakpoint resume, and output generation instructions. Covers PROC-05, PROC-07, PROC-08, OUT-01, OUT-05.

Section 1: Batch LLM Processing
- Collect all items from today's data/news/YYYY-MM-DD.jsonl with processing_status "raw"
- Group into batches of 5-10 items
- For each batch:
  1. Read references/prompts/classify.md, fill {categories_list} from config/categories.json, fill {news_batch} with batch items (id, title, source, content snippet)
  2. Process the prompt (agent performs classification using the template)
  3. Parse JSON array response, match each result to its item by id
  4. Update item fields: categories.primary, categories.tags, importance_score, form_type
  5. Read references/prompts/summarize.md, fill {news_batch} with same items
  6. Process the prompt (agent performs summarization)
  7. Parse JSON array response, update item.content_summary
  8. Mark processing_status "complete" for successfully processed items
  9. Update config/budget.json: increment calls_today by 2 (one classify + one summarize call per batch), estimate and increment tokens_today

Section 2: Error Handling (from gpt-plan-v3.md Section 4.2)
- LLM call failure: skip classification/summarization, mark processing_status "partial", item appears in digest with title + source only
- LLM format error (invalid JSON): retry once with same prompt. If still fails, mark "partial"
- Classify fails but summarize succeeds: item enters digest in exploration slot, primary category set to null
- Budget check: before each batch, read budget.json. If calls_today >= daily_llm_call_limit, stop processing. If calls_today >= alert_threshold * daily_llm_call_limit, log warning but continue
- Date rollover: if budget.json current_date != today, reset calls_today and tokens_today to 0, update current_date

Section 3: Breakpoint Resume (PROC-08)
- At start of processing phase: scan today's JSONL for items with processing_status "raw" or "partial"
- "raw" items: full classify + summarize
- "partial" items: attempt only the missing step (if categories present but summary missing, only summarize; if neither, full process)
- This ensures interrupted runs are completed on next trigger without reprocessing already-complete items

Section 4: Output Generation (OUT-01, OUT-05)
- Read references/scoring-formula.md and compute final_score for each item with processing_status "complete" or "partial"
- For "partial" items: use defaults (importance_score 0.3, primary category null -> exploration)
- Sort by final_score descending
- Apply quality gate:
  - 0 scored items: do NOT generate output. Log "no content for digest" in metrics. Release lock and exit.
  - 1-2 items: generate shortened digest with only "Core Focus" section
  - 3-14 items: generate digest with available sections, skip empty sections
  - 15+ items: full digest per output-templates.md (core 50%, adjacent 20%, hotspot 15%, exploration 15% -- approximate, not strict for MVP)
- Read references/output-templates.md for Markdown format
- Generate digest, write to output/latest-digest.md (atomic write)
- Write daily metrics to data/metrics/daily-YYYY-MM-DD.json (schema from references/data-models.md)
- Release file lock: delete data/.lock

**scripts/dedup-index-rebuild.sh** -- Shell script to rebuild dedup-index.json from JSONL files. Used for recovery or maintenance.
```bash
#!/bin/bash
# Rebuild dedup-index.json from recent JSONL files (last 7 days)
# Usage: bash scripts/dedup-index-rebuild.sh [base_dir]

BASE_DIR="${1:-.}"
NEWS_DIR="$BASE_DIR/data/news"
INDEX_FILE="$NEWS_DIR/dedup-index.json"
TEMP_FILE="$INDEX_FILE.tmp.rebuild"

echo "{}" > "$TEMP_FILE"

# Process JSONL files from last 7 days
for f in $(find "$NEWS_DIR" -name "*.jsonl" -mtime -7 2>/dev/null | sort); do
  while IFS= read -r line; do
    id=$(echo "$line" | python3 -c "import sys,json; print(json.load(sys.stdin).get('id',''))" 2>/dev/null)
    source_id=$(echo "$line" | python3 -c "import sys,json; print(json.load(sys.stdin).get('source_id',''))" 2>/dev/null)
    fetched_at=$(echo "$line" | python3 -c "import sys,json; print(json.load(sys.stdin).get('fetched_at',''))" 2>/dev/null)
    if [ -n "$id" ]; then
      python3 -c "
import json, sys
idx = json.load(open('$TEMP_FILE'))
idx['$id'] = {'news_id': '$id', 'source_id': '$source_id', 'fetched_at': '$fetched_at'}
json.dump(idx, open('$TEMP_FILE', 'w'), ensure_ascii=False, indent=2)
"
    fi
  done < "$f"
done

mv "$TEMP_FILE" "$INDEX_FILE"
echo "Rebuilt dedup-index with $(python3 -c "import json; print(len(json.load(open('$INDEX_FILE'))))" 2>/dev/null || echo 0) entries"
```
