# Milestone v2.0: Quality & Robustness

**Status:** SHIPPED 2026-04-03

## Overview

`v2.0` turned the v1.0 MVP into a more trustworthy and operable skill. The milestone focused on documentation, localization, deterministic contracts, lower-noise processing, alert-fatigue reduction, observability, and user/operator interaction quality.

## Milestone Summary

**Key Decisions:**

- Keep user-facing rendering fully Chinese while preserving internal English identifiers in contracts and schemas.
- Use `prompt_version` as the cache invalidation primitive.
- Derive recommendation evidence from deterministic scoring and quota state, not from generated rationale.
- Keep alert governance in `AlertState` and digest repeat history in `DigestHistory` rather than overloading `DailyMetrics`.
- Treat schema/version registry updates and fixtures as first-class parts of prompt/config work.

**Issues Resolved:**

- Repo onboarding and deployment documentation gap
- Output-language inconsistency and raw-field rendering leaks
- Stale cache reuse after prompt changes
- Low-value item spend before classification
- Alert fatigue from repeated or low-context updates
- Missing operator-grade diagnostics and source inspection workflows

**Issues Deferred:**

- Live platform smoke testing for cron delivery, isolated sessions, exec permissions, and timeout behavior
- Hardening follow-up items in the `HARD-*` backlog
- Nyquist validation coverage completion for milestone phases

**Technical Debt Incurred:**

- Several validation docs are still missing or remain in draft/non-compliant state
- Planning docs drifted during execution and had to be reconciled during milestone closure
