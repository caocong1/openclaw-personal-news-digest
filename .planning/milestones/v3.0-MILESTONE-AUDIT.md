## Integration Findings

### CRITICAL: `data/provenance/` Directory Missing

**Files affected:** `data/provenance/provenance-db.json`, `data/provenance/citation-graph.json`, `data/provenance/tier-stats.json`, `data/provenance/provenance-discrepancies.jsonl`, `data/provenance/discovered-sources.json`

**Evidence:** `ls data/provenance/` returns "No such file or directory"

SKILL.md Provenance Phase step 6 documents writes to all five files. Phase 14 Source Discovery Phase reads from `discovered-sources.json`. Phase 15 joins selected items to `provenance-db.json` for scoring and rendering. The Bootstrap step (Collection Phase step 0) creates directories but `data/provenance/` is not listed.

**Impact:** Every provenance wire from Phase 13 to Phase 14/15 is a dead read at runtime. Scoring cannot look up provenance records (PIPE-01). Digest cannot render tier attribution (PIPE-04). Source discovery has no persistent state (DISC-01). Weekly report receives empty data (PIPE-05). PROV-06 cannot reconstruct provenance chains.

**Fix:** Create `data/provenance/` directory. Touch empty `provenance-db.json` (`[]` or `{}`), `citation-graph.json` (`{"nodes":[],"edges":[]}`), `tier-stats.json` (`{"_schema_v":1,"days":{}}`), `discovered-sources.json` (`{"_schema_v":1,"domains":[]}`), and `provenance-discrepancies.jsonl` (empty file). Add `data/provenance/` to the Bootstrap step directory list in SKILL.md Collection Phase step 0.

### HIGH: `backlog_tools.append_failure_followup` Not Called from SKILL.md

**File affected:** SKILL.md
**Evidence:** `grep -c "backlog_tools\|append_failure_followup" SKILL.md` returns 0

16-03-PLAN.md task 3 explicitly calls for "Update SKILL.md to call `backlog_tools.append_failure_followup` when any journal entry with severity=error is written." The module `scripts/lib/backlog_tools.py` exists and implements the function correctly. The Operational Rules section (Rule 4) describes the failure_type mapping. But SKILL.md never actually calls it.

**Impact:** OPER-03 is implemented at the module layer but never wired into the pipeline. Every error journal entry creates a journal record but no backlog follow-up entry.

**Fix:** After each `run-journal.sh append ... error ...` call in SKILL.md, add a corresponding `bash scripts/run-journal.sh backlog "$RUN_ID" ...` call with the failure_type mapping applied: `SRC_TIMEOUT` → `"source_timeout"`, `LLM_FAILURE` → `"llm_failure"`, `SRC_MALFORMED` → `"degraded_sources"`.

### MEDIUM: E2E Fixture File Missing

**File:** `data/fixtures/provenance-ranking-e2e-sample.json`
**Evidence:** File referenced in 15-03-PLAN.md and verified in 15-VERIFICATION.md but not found on disk.

**Impact:** PIPE-02 smoke test (alert threshold correctness) cannot run without the fixture. Behavioral assertions in 15-VERIFICATION.md cannot be re-executed.

**Fix:** Create the fixture file with all five PIPE proof sections (PIPE-01 provenance scoring, PIPE-02 alert gating, PIPE-03 representative selection, PIPE-04 provenance rendering, PIPE-05 weekly discovery report).

### LOW: Phase 16 VERIFICATION vs MILESTONE-AUDIT Inconsistency

Phase 16 VERIFICATION.md reports "12/12 truths verified" because it checks artifact existence and internal consistency. The milestone audit found that two artifacts (backlog_tools module, data/provenance/ directory) are present but not wired into SKILL.md at runtime. Phase-level verification and integration-level verification use different scopes.
