# Requirements Archive: v2.0 Quality & Robustness

## Requirement Outcomes

All requirements planned for `v2.0 Quality & Robustness` were satisfied. A few rows in the live milestone `REQUIREMENTS.md` lagged behind execution, but phase verification and summary evidence confirmed the underlying work was complete, and the archive reflects the final reconciled state.

## v2.0 Requirements

### Documentation

- [x] **DOC-01**: Project root has `README.md` with architecture, deployment instructions, configuration guide, and operational scripts documentation

### Localization

- [x] **L10N-01**: Daily digest output uses Chinese labels for all section headers and metadata
- [x] **L10N-02**: Breaking alert output uses Chinese labels
- [x] **L10N-03**: Weekly report output uses Chinese labels for all sections
- [x] **L10N-04**: Summarize and weekly-report prompts enforce Chinese-output rule

### Data Quality

- [x] **QUAL-01**: Pre-write quality contract validates UTF-8 sanitization and title validation for all source types
- [x] **QUAL-02**: Quick-Check output strips JSON field names from alert rendering
- [x] **QUAL-03**: Output templates define rendering contract separating user-facing vs internal fields

### Infrastructure

- [x] **INFRA-01**: Cache entries include `prompt_version`; version mismatch triggers cache miss
- [x] **INFRA-02**: Data models include bootstrap and migration notes with a new-fields registry
- [x] **INFRA-03**: `SKILL.md` verifies required directories on first run
- [x] **INFRA-04**: Deterministic fixture directory exists for verification scenarios

### Noise Filtering

- [x] **NOISE-01**: Pre-classify noise filter skips items matching source `noise_patterns`
- [x] **NOISE-02**: Post-classify filter marks items with `importance < 0.25` as `digest_eligible: false`
- [x] **NOISE-03**: Filtered items stay queryable in JSONL history
- [x] **NOISE-04**: Source schema supports `noise_patterns` and `title_discard_patterns`
- [x] **NOISE-05**: `DailyMetrics` tracks `noise_filter_suppressed`

### Classification

- [x] **CLASS-01**: Classification prompt includes stronger low-end calibration, borderline examples, and disambiguation rules
- [x] **CLASS-02**: Category config supports `negative_examples` and prompt assembly includes them
- [x] **CLASS-03**: Classification cache version bumped from `classify-v1` to `classify-v2`

### Alert System

- [x] **ALERT-01**: Daily alert-state file enforces a 3-alert daily cap and URL dedup
- [x] **ALERT-02**: Event objects store per-event alert memory
- [x] **ALERT-03**: Quick-Check uses a unified alert decision tree
- [x] **ALERT-04**: Delta alerts fire for qualifying event updates
- [x] **ALERT-05**: Delta alerts show what changed vs the previous alert
- [x] **ALERT-06**: Standard-alert fallback exists when event context is unavailable

### Dedup

- [x] **DEDUP-01**: `DigestHistory` tracks the last five runs with timeline snapshots
- [x] **DEDUP-02**: Cross-digest repetition penalty applies to stale events
- [x] **DEDUP-03**: Output footer shows suppressed repeat-item counts

### Observability

- [x] **OBS-01**: `source_count` reflects enabled sources and failed sources are named in the footer
- [x] **OBS-02**: `DailyMetrics` includes a populated `run_log`
- [x] **OBS-03**: Schema Version Registry documents current versions and change history
- [x] **OBS-04**: Diagnostics command reads metrics, alert state, and digest history

### Interaction

- [x] **INTERACT-01**: Scheduling profiles are configurable via repo-backed skill commands
- [x] **INTERACT-02**: Source-status command shows per-source health and enable/disable state
- [x] **INTERACT-03**: Recommendations include structured evidence for why items were selected
- [x] **INTERACT-04**: NL intent recognition lives in `feedback-rules.md` without duplication in `SKILL.md`
- [x] **INTERACT-05**: Rolling coverage collapses events with more than five same-day items into a timeline view
